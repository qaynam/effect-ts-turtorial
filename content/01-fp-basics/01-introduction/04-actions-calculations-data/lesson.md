副作用はなくせない、でも「いつどこで起こるか分からない」のは困る。前のレッスンのこの結論から、実践的な指針が出てきます。コードを次の 3 種類に仕分けるという考え方です。

- **アクション**：呼ぶタイミングや回数が結果に影響するコード。`console.log`、通信、ファイル書き込みなど、副作用を含むものすべて
- **計算**：入力から出力を求めるだけの純粋関数。いつ、何回呼んでもよい
- **データ**：ただの値。文字列、数値、配列、オブジェクト

方針はシンプルで、**計算をできるだけ厚くし、アクションをプログラムの端に薄く追い出す**ことです。テストしやすく再利用しやすい計算が本体になり、予測しにくいアクションは境界に並ぶだけになります。

## 混ざったコードの不便さ

右の `printTotal` は、合計を求める計算と、表示するアクションが 1 つの関数に混ざっています。このままでは「送料込み: 5500円」も表示したいと思っても、`printTotal` を再利用できません。合計を**値として**受け取る手段がないからです。

## 計算を抽出する

まず、計算部分だけを取り出した `calculateTotal` を作りましょう。表示はせず、合計を返すだけです。

```ts
+++function calculateTotal(items: number[]): number {
  let total = 0
  for (const item of items) {
    total += item
  }
  return total
}+++
```

次に、`printTotal` を削除し、アクション(`console.log`)はプログラムの端でまとめて行います。計算結果はただの数値なので、送料の 500 円を足すのも自由自在です。

```ts
~~~printTotal(cart)~~~
+++console.log(`合計: ${calculateTotal(cart)}円`)
console.log(`送料込み: ${calculateTotal(cart) + 500}円`)+++
```

Run して 2 行表示されれば完成です。`calculateTotal` は何回呼んでも安全なので、同じ関数を 2 回呼ぶことにためらいがありません。Part 3 で学ぶ Effect は、この分類の「アクション」を値として持ち運べるようにする道具です。この仕分けの感覚は最後まで使い続けます。
