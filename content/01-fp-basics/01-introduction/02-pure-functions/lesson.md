前のレッスンで、FP は「コードを予測可能にする考え方」だと言いました。その予測可能性を支えるのが**純粋関数**(pure function)です。純粋関数とは、次の 2 つを満たす関数のことです。

- 同じ入力には、必ず同じ出力を返す
- 外部に影響を与えない(外側の変数を読まない・書き換えない)

この 2 つを満たす関数は、いつ呼んでも、何回呼んでも、結果が引数だけで決まります。テストは「入力を渡して出力を見る」だけで済み、呼び出す順番を入れ替えても壊れません。

## 不純な関数を観察する

右の `greet` は、引数を取らず、外側の変数 `name` を読んでいます。まず Run してみてください。`greet()` という**まったく同じ呼び出し**なのに、間で `name` が書き換えられたせいで、1 回目と 2 回目の結果が違います。この関数の結果を予測するには、コード全体で `name` に何が起きるかを追いかけるしかありません。

## 純粋にする

必要なものをすべて引数で受け取るように直しましょう。

まず、`greet` が `name` を引数として受け取るようにします。

```ts
function greet(+++name: string+++): string {
  return `Hello, ${name}!`
}
```

次に、外側の `let name` と再代入を消し、呼び出すときに値を渡します。同じ入力なら同じ出力になることを確かめたいので、2 回とも `"World"` を渡してください。

```ts
~~~let name = "World"~~~

console.log(greet(+++"World"+++))
~~~name = "Effect"~~~
console.log(greet(+++"World"+++))
```

Run すると、今度は 2 行とも `Hello, World!` になります。`greet` の結果は引数だけで決まり、まわりで何が起きても変わりません。では「外部に影響を与えるコード」を完全になくせるのかというと、そうはいきません。それが次のレッスンの主題、副作用です。
