前のレッスンの「外部に影響を与える(または外部に左右される)」に、名前を付けましょう。関数が戻り値を返す以外に行うこと、それが**副作用**(side effect)です。身近な例はどれも副作用です。

- `console.log`：画面という外部の状態を変える
- `Date.now()` や `Math.random()`：呼ぶたびに違う値を外部から読む
- 引数の破壊的変更：渡された配列やオブジェクトを書き換える

副作用そのものが悪なのではありません。画面に何も表示せず、何も保存しないプログラムは役に立たないからです。悪いのは、**副作用がいつどこで起こるか分からない**ことです。中でもタチが悪いのが引数の破壊的変更で、呼び出した側は自分の持っている配列が変わったことに気づけません。

## 気づかないうちに壊れる

右の `top3` は、スコアの上位 3 件を返す関数です。Run してみてください。1 行目は期待どおりですが、2 行目を見ると、元の配列 `scores` の先頭が `50` から `100` に変わってしまっています。`sort` が配列を**その場で**並べ替える(破壊的な)メソッドだからです。

## 非破壊に直す

コピーを作ってからソートすれば、元の配列はそのまま残ります。引数なしの `slice()` は配列全体のコピーを返すので、これを挟みましょう。

```ts
function top3(scores: number[]): number[] {
  return scores+++.slice()+++.sort((a, b) => b - a).slice(0, 3)
}
```

Run すると、2 行目が `50` のまま変わらなくなります。`top3` は純粋関数になり、何回呼んでも呼び出し側の世界に影響しません。この「変更せず、新しく作る」というテクニックは、次の章のイミュータビリティのレッスンでさらに広げます。
