前のレッスンで、関数は「生まれた場所の環境を覚えた値」だと分かりました。この章では、その性質を使って関数を組み立てる技法を 3 つ学びます。1 つめが**カリー化**(currying)です。複数の引数を取る関数を、「1 引数関数を返す 1 引数関数」の連なりに変換することを指します。

```ts
const add = (a: number, b: number) => a + b            // 2 引数を一度に受け取る
const addCurried = (a: number) => (b: number) => a + b // 1 引数ずつ受け取る
addCurried(2)(3) // 5
```

`addCurried(2)` の時点では、まだ足し算は起きません。返ってくるのは「`a` が 2 だと覚えている関数」、つまり前のレッスンで学んだクロージャです。残りの `(3)` を渡して、はじめて計算が完了します。

## 課題: addCurried を完成させる

エディタの `addCurried` は形だけできていて、中身が仮実装の `0` のままです。覚えている `a` と、あとから来る `b` を足すように直しましょう。

```ts
const addCurried = (a: number) => (b: number): number => ~~~0~~~+++a + b+++
```

Run すると 3 行出力されます。`addCurried(2)(3)` は `add(2, 3)` と同じ 5。そして 3 行目に注目してください。`addCurried(10)` と**途中で止めて**作った `add10` が、あとから `5` を受け取って 15 を返しています。2 引数の `add` にはできない芸当です。

エディタで `addCurried` にマウスカーソルを乗せてみましょう。型が `(a: number) => (b: number) => number` と表示され、「1 引数関数を返す 1 引数関数」がそのまま読み取れます。この「途中で止める」が何の役に立つのかは、次のレッスンで分かります。
