`myMap` は「配列の形を保ったまま、各要素を変換する」道具でした。要素 3 個を渡せば、返ってくるのも 3 個です。ではデータ処理のもう一方、**たくさんの値をひとつの値にまとめる**ほうはどうでしょう。合計、最大値、件数、文字列の連結。書いてみると、どれも骨組みがそっくりです。

```ts
let total = 0                    // 初期値がある
for (const n of numbers) {
  total = total + n              // 要素を 1 つずつ取り込んで累積する
}
return total                     // 最後の累積値が答え
```

`total = total + n` を `total = Math.max(total, n)` に変えれば最大値、文字列に変えれば連結です。つまり違うのは**初期値**と**累積のしかた**だけ。この骨組みを取り出したものを**畳み込み**(fold)と呼びます。

## 課題: myFold を完成させる

前回と同じ要領で、変わる部分を関数として受け取りましょう。エディタの `myFold` は、要素の型 `A` と結果の型 `B` を受け取り(合計なら `A` も `B` も数値、連結なら `A` は文字列で `B` も文字列)、`initial` から始めて累積関数 `f` で畳み込みます。まだ `f` を使っていないので、累積するように直してください。

```ts
  for (const item of items) {
    acc = ~~~acc~~~+++f(acc, item)+++
  }
```

Run すると、**同じ `myFold`** が合計 `10`、最大値 `9`、連結 `abc` の 3 役をこなします。ここでも種明かしをすると、これは `Array.prototype.reduce` の再発明です。`map` と同じく、標準ライブラリの便利メソッドは高階関数として組み立てられています。

## 発展: map を fold で書く

fold の表現力は、実は map より広いです。空配列を初期値にして、変換した要素を足しながら畳み込めば、それはもう map です。

```ts
+++function myMap<A, B>(items: A[], g: (item: A) => B): B[] {
  return myFold(items, [] as B[], (acc, item) => [...acc, g(item)])
}

console.log(myMap([1, 2, 3], (n) => n * 2).join(","))+++
```

`2,4,6` が出れば成功です。`acc` を書き換えず新しい配列を作っているので、前々回のイミュータビリティもそのまま守られています。

「初期値と、値を 1 つずつ取り込む関数」という形は、配列に限りません。Part 2 では、値があるかもしれない `Option` を畳み込んで既定値を取り出す `getOrElse` が、Part 3 では副作用を含む処理を順に畳み込む `Effect.reduce` が出てきます。
