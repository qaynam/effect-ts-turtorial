「map を持つ箱が Functor」と言いましたが、実は条件がもう少しあります。その `map` は、次の 2 つの法則を満たしていなければなりません。

- **恒等則**: `map(box, id)` は `box` と同じ。何もしない関数を map しても、箱は何も変わらない
- **合成則**: `map(map(box, f), g)` は `map(box, (a) => g(f(a)))` と同じ。2 回に分けて map しても、合成してから 1 回 map しても、結果は変わらない

数学の定理のように見えますが、証明は要りません。自作した `map` が実際に満たしているか、**コードを動かして確かめられる**からです。右のコードには、恒等則の検証がすでに書いてあります。同じ要領で残りを書きましょう。

## Option の合成則を検証する

`f`(+1)してから `g`(×2)する 2 通りの書き方を比べます。

```ts
+++const twice = map(map(some(5), f), g)
const once = map(some(5), (n) => g(f(n)))
console.log(`合成則: ${show(twice)} = ${show(once)}`)+++
```

## Array でも検証する

同じ法則が、まったく別の実装である `Array.prototype.map` でも成り立つことを確かめます。

```ts
+++const chained = [1, 2, 3].map(f).map(g).join(",")
const fused = [1, 2, 3].map((n) => g(f(n))).join(",")
console.log(`Array の合成則: ${chained} = ${fused}`)+++
```

Run して、3 行とも `=` の左右が一致していれば検証成功です。

## 法則はリファクタリングの保証書

合成則の 2 つの式をもう一度見てください。左は「map を 2 回」、右は「map を 1 回」。つまりこの法則は、**`.map(f).map(g)` を `.map(g ∘ f)` に書き換えても意味が変わらない**ことを保証しています。ループを 2 周から 1 周に減らす最適化や、逆に長い関数を小さな map の連鎖に分解する整理を、動作を壊す心配なしに行えるわけです。

法則と聞くと縛りのようですが、実際にはその逆で、「この箱はいつもの map と同じように振る舞う。安心して式変形してよい」という保証書です。今後登場する Functor(Effect を含む)は、すべてこの保証書付きです。
